FCFS — First Come First Served NO-PREEMPTIVE
Ejecuta procesos en orden de llegada. El primero que llega se ejecuta hasta completarse. Simple pero puede causar el efecto convoy.
PROCESS TABLE
PIDARRIVALBURST
ALGORITHM INFO
// Ordenar por Arrival Time
CT = tiempo_fin
TAT = CT - Arrival_Time
WT = TAT - Burst_Time
RT = primera_CPU - Arrival_Time
// Complejidad: O(n log n)
// No hay starvation
// WT_avg puede ser alto
SJN — Shortest Job Next NO-PREEMPTIVE
De los disponibles elige el de menor Burst Time. Minimiza WT promedio. Puede causar starvation en procesos largos.
PROCESS TABLE
PIDARRIVALBURST
PSEUDOCODE
while procs_pendientes:
  avail = [p for p in cola
           if p.AT <= tiempo]
  if avail:
    next = min(avail,
           key=p.BT)
    tiempo += next.BT
    calc(next)
  else:
    tiempo = next_arrival()
Round Robin PREEMPTIVE
Cada proceso recibe un quantum. Si no termina, regresa al final de la cola. Justo, buen tiempo de respuesta interactivo.

SRT — Shortest Remaining Time PREEMPTIVE
Versión expulsiva de SJN. Nuevo proceso con BT < remaining(actual) → expulsa al actual. Óptimo para minimizar TAT promedio.
PROCESS TABLE
PIDARRIVALBURST
SJN vs SRT
SJN → decide al inicio del burst
SRT → evalúa en cada evento
IF new.BT < cur.remaining:
  PREEMPT(cur)
  RUN(new)
// Starvation posible
HRRN — Highest Response Ratio Next NO-PREEMPTIVE
Selecciona el proceso con mayor RR = (WT + BT) / BT. Evita starvation porque el ratio crece con el tiempo de espera.
PROCESS TABLE
PIDARRIVALBURST
RESPONSE RATIO
RR = (WT + BT) / BT
WT = t_actual − AT − t_ejecutado
BT = burst_time_original
// RR siempre >= 1.0
// Mayor espera → mayor RR
// Anti-starvation inherente
