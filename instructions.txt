Instructions
Es fundamental que presten atención a la validación del desarrollo de su proyecto final, ya que no basta con una simulación visual o lógica de los algoritmos y procesos. El sistema simulador debe implementar de manera explícita el uso de threads, así como considerar y evidenciar el uso de paralelismo, multithreading, concurrencia y la simulación de múltiples cores de CPU. Esto implica que los procesos no solo se representen gráficamente, sino que también permita visualizar threads de forma concurrente dentro del simulador, distribuyéndose entre cores y reflejando el comportamiento real de un sistema operativo moderno. La correcta implementación de estos elementos será un criterio clave para la evaluación del proyecto.

· Uso de multiprocesamiento mediante fork
· Uso de multithreading (threads)
· Ejecución en paralelismo real o simulado
· Simulación explícita de múltiples cores de CPU

debe ejecutarse utilizando threads y/o forks, no únicamente simulación lógica de procesos.

Cada proceso del simulador debe:

· Ejecutarse como thread o fork
· Poder ejecutarse en paralelo
· Poder asignarse a un core o varios cores dependiendo de los INPUT
